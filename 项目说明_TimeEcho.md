# 项目说明：时光回声（TimeEcho）

> **⚠️ 方向推断说明**
> 本文档基于比赛主题"From Me to We"和老师示范"与过去/未来自己对话"推断得出。
> 如果你心里有不同方向，直接在对应部分批注修改即可。

---

## 1. 产品一句话定位

**写给未来自己的信，AI 帮你生成回信——同时让你发现：原来别人也在问同样的问题。**

从"Me"（我和自己的时间对话）到"We"（发现一群人共享同样的困惑与期待）。

---

## 2. 目标用户和真实问题

**目标用户**：18-30 岁年轻人（大学生、刚工作的人），正处在人生选择、迷茫、转型的阶段。

**真实问题**：
- 很多话想对自己说，但写出来觉得矫情，不知道怎么表达
- 想知道"未来的我会怎么看现在的困惑"，但没有人能回答
- 觉得自己的迷茫是孤独的，不知道别人其实也在经历类似的事

**情绪锚点**：孤独感 + 对未来的好奇 + "我不是一个人"的慰藉

---

## 3. 用户完成一次任务的 3 步流程

```
第 1 步：写一句话给未来的自己
        → "三年后的我，我现在不知道要不要休学"
        → "五年后的我，你过得开心吗"

第 2 步：AI 生成未来自己的回信
        → 基于用户输入的语气、年龄、情境，生成一封有温度的回信
        → 页面展示回信，带有打字机效果

第 3 步：看见"回声"
        → 页面下方浮现 2-3 条其他人问过类似问题的对话（匿名）
        → 配文："你不是唯一在问这个问题的人"
        → 从"Me"变成"We"
```

---

## 4. MVP 只做的 3 个功能

| 功能 | 说明 |
|------|------|
| **写信 + AI 回信** | 用户输入一句话 → AI 生成未来自己的回信（300 字以内，有情感、不空洞） |
| **回声匹配** | 用语义相似度匹配其他人的提问，展示 2-3 条匿名对话 |
| **视觉氛围** | 整体视觉是"时间 / 信件 / 星空"的氛围，打字机动画 + 微光效果 |

## 7. AI 需要完成的任务

### 任务 1：生成回信
- **输入**：用户写的一句话 + 可选的"几年后"选择（默认 3 年）
- **Prompt 方向**：
  - 你是这个用户 N 年后的自己
  - 用第二人称"你"称呼过去的自己
  - 语气温暖但不鸡汤，像一个真实的人在说话
  - 300 字以内
  - 不要说教，不要用"亲爱的"开头
  - 可以承认"我现在也还在摸索"
- **输出**：一封信

### 任务 2：语义匹配回声
- **输入**：用户写的那句话
- **处理方式（MVP 阶段）**：
  - 方案 A：调用 embedding API 计算相似度，匹配预置语料库
  - 方案 B：直接让 LLM 判断"以下哪些问题和用户的问题在情感主题上最接近"，从预置语料中选 2-3 条
- **输出**：2-3 条匿名对话（问题 + 回信摘要）

## 8. 第一阶段开发步骤

1. **搭页面骨架**
   - 单 HTML 文件，内联 CSS + JS
   - 标题 + 输入框 + 按钮 + 回信展示区 + 回声展示区

2. **接入 AI 回信**
   - 前端直接调用 AI API（用 fetch）
   - 打字机效果逐字显示回信

3. **预置回声语料**
   - 手写 15-20 条"其他人"的提问 + 回信，存在 JS 数组里
   - 覆盖常见主题：学业迷茫、职业选择、感情、自我怀疑、家庭压力

4. **回声匹配**
   - 用 AI 做语义匹配，从预置语料中选出 2-3 条

5. **视觉打磨**
   - 深色背景 + 微光 + 打字机动画
   - 回声卡片渐入效果

6. **自测**
   - 输入不同类型的问题，确认回信质量和回声匹配合理
   - 确认全流程 30 秒内完成

## 9. 需要 mock 的数据

### 预置回声语料（15-20 条）

每条格式：
```json
{
  "id": "echo_01",
  "question": "毕业了要不要回老家",
  "reply_summary": "三年后我回来了，发现回来不是认输，是终于知道自己要什么了",
  "tags": ["职业", "家庭", "选择"]
}
```

建议覆盖主题：
- 学业迷茫（休学 / 转专业 / 考研）
- 职业选择（大厂 vs 小公司 / 回老家 vs 留大城市）
- 感情（分手 / 表白 / 异地）
- 自我怀疑（我是不是不够好 / 别人都比我强）
- 家庭（和父母的关系 / 家人的期望）
- 存在感（活着的意义 / 我是谁）

## 10. 后续可能接入的 API

| API | 用途 | 优先级 |
|-----|------|--------|
| AI 对话 API（如 GPT / Claude / 文心） | 生成回信 + 语义匹配 | P0 必须 |
| Embedding API | 更精准的回声语义匹配 | P1 可选 |
| 用户内容存储（如 Supabase） | 让回声库随用户使用自动增长 | P2 后续 |
| 分享 / 截图生成 API | 生成回信卡片图供分享 | P2 后续 |

## 技术选型建议（给 Coding Agent 参考）

- **前端**：纯 HTML + CSS + JS，不需要框架
- **AI 调用**：前端 fetch 直接调 API，MVP 阶段不需要后端
- **部署**：静态文件，可部署到任何静态托管
- **字体**：中文用思源宋体 / 霞鹜文楷（有信件感），英文用 serif
- **配色**：深蓝/深紫底 + 暖黄/暖白文字（夜空 + 灯火的氛围）
